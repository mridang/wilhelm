# Contributing to Wilhelm

Thank you for your interest in contributing!

## Before you start

- Open an issue first for non-trivial changes so the approach can be agreed on
  before you invest time writing code.
- For typo fixes, doc improvements, and small bug fixes a PR without an issue
  is fine.

## Development setup

```bash
# Install devbox (https://www.jetify.com/devbox/)
devbox shell          # activates Go 1.26, golangci-lint, dprint, etc.
make build            # build root module
make test             # run unit tests
make lint             # golangci-lint
make format           # dprint + gofmt
```

See the [README](README.md) for a full list of `make` targets.

## Adding a new CRD family

```bash
bash hack/scaffold-crd.sh <family-path> <version> <upstream-import-path> <api-group>
# e.g.:
bash hack/scaffold-crd.sh argo/cd v1alpha1 \
  github.com/argoproj/argo-cd/v2/pkg/apis/application/v1alpha1 \
  argoproj.io
```

The script scaffolds both `assert/<family>/` and `env/<family>/` modules,
regenerates code, and updates `go.work`. Open a PR with the result.

## Pull request checklist

- [ ] `make test` passes
- [ ] `make lint` is clean
- [ ] `make format` produces no diff
- [ ] `bash hack/verify-all.sh` passes for every affected module
- [ ] New CRD families include a README generated by `hack/gen-crd-readmes.sh`
- [ ] Commit messages follow [Conventional Commits](https://www.conventionalcommits.org/)
  (`feat:`, `fix:`, `docs:`, `chore:` etc.) — semantic-release uses these to
  determine the next version

## Code style

- Standard `gofmt`/`goimports` formatting (enforced by CI)
- All exported symbols must have a doc comment
- Generated files are named `zz_generated.go` — do not edit them by hand;
  run `go generate ./...` instead
